Cụ thể: 🧩 1. Dán vào GitHub Dán các file sau vào repository trên GitHub (ví dụ: SupplyApply-Dashboard): File Vị trí Ghi chú index.html (hoặc Index.html) gốc repo Đây là giao diện chính — chứa HTML, CSS, JS của bạn style.css (nếu có) cùng thư mục hoặc trong /assets/css/ CSS tách riêng nếu bạn không nhúng trực tiếp script.js (nếu có) cùng thư mục hoặc trong /assets/js/ JavaScript tách riêng nếu không viết inline trong HTML .gitignore gốc repo (tuỳ chọn) để bỏ qua file cục bộ như .DS_Store README.md gốc repo (tuỳ chọn) mô tả dự án, ví dụ: cách deploy lên Cloudflare ☁️ 2. KHÔNG dán vào GitHub Các file sau không cần hoặc không nên đưa lên GitHub: File Lý do Code.gs (Apps Script) Đây là backend chạy trên Google Script — không chạy được qua GitHub credentials.json, token.json Thông tin bảo mật — tuyệt đối không commit File .zip hoặc backup cũ Không cần thiết cho build frontend 🚀 3. Cloudflare Pages tự build Khi Cloudflare Pages nhận repo, nó sẽ tự nhận index.html làm trang chính. Sau khi push code, Cloudflare sẽ cấp URL dạng: https://.pages.dev 👉 Tóm lại: Chỉ cần dán index.html (và các file CSS/JS tĩnh) vào GitHub — KHÔNG cần Code.gs.
